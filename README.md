# ТЕХНИЧЕСКОЕ ЗАДАНИЕ
## на разработку тренажёра-стенда для обучения распознаванию фишинговых атак

### 1. Основание для разработки
Разработка учебного тренажёра обусловлена необходимостью повышения уровня киберграмотности сотрудников и снижения рисков, связанных с фишинговыми атаками в организации.

### 2. Назначение разработки
Создание программного комплекса, имитирующего работу фишинговых сайтов для проведения учебных тренировок сотрудников с последующей статистической обработкой результатов.

### 3. Цели разработки
- Создание реалистичной имитации фишинговых веб-сайтов
- Реализация системы сбора и анализа статистики по действиям пользователей
- Разработка интерактивной системы обратной связи и обучения
- Создание инструмента для оценки эффективности обучения кибербезопасности

### 4. Требования к функциональным характеристикам
#### 4.1. Симуляция фишинговых атак
- Имитация популярных социальных сетей (ВКонтакте и аналоги)
- Реализация сценариев фишинга с различными уровнями сложности
- Симуляция антивирусных предупреждений

#### 4.2. Система мониторинга и статистики
- Запись всех действий пользователей
- Анализ успешности распознавания фишинга
- Генерация отчетов в реальном времени
- Поддержка нескольких параллельных тренажёров

#### 4.3. Обучающая система
- Интерактивные уроки по кибербезопасности
- Персонализированные рекомендации
- Визуальное сравнение фишинговых и легитимных сайтов

### 5. Требования к надежности
- Гарантированное сохранение статистики
- Восстановление после сбоев
- Резервное копирование данных

### 6. Условия эксплуатации
- Веб-доступ через стандартные браузеры
- Поддержка мобильных устройств
- Работа в корпоративной сети

### 7. Требования к составу и параметрам технических средств
- Сервер с ОС Linux/Windows Server
- СУБД PostgreSQL 12+
- Веб-сервер Apache/Nginx
- PHP 7.4+

### 8. Требования к информационной и программной совместимости
- Соответствие стандартам HTML5, CSS3
- Поддержка современных браузеров
- Совместимость с корпоративными системами аутентификации

### 9. Стадии и этапы разработки
1. Проектирование архитектуры
2. Разработка ядра системы
3. Создание интерфейсов тренажёров
4. Реализация системы статистики
5. Тестирование и отладка
6. Внедрение и обучение администраторов

### 10. Порядок контроля и приемки
- Функциональное тестирование всех модулей
- Проверка безопасности системы
- Приемочные испытания с участием конечных пользователей
- Анализ соответствия требованиям ТЗ

# ОПИСАНИЕ ПРОГРАММЫ
## "Тренажёр-стенд для обучения распознаванию фишинговых атак"

## 1. Общие сведения

### 1.1. Обозначение и наименование программы
- **Обозначение:** PHISH-TRAINER/1.0  
- **Наименование:** "Тренажёр-стенд для обучения распознаванию фишинговых атак"  
- **Краткое наименование:** Фишинг-тренажёр

### 1.2. Программное обеспечение, необходимое для функционирования
- Веб-сервер Apache 2.4+ или Nginx 1.18+
- СУБД PostgreSQL 12+
- Интерпретатор PHP 7.4+ с расширениями:
  - pdo_pgsql
  - session
  - json
- Веб-браузер с поддержкой HTML5, CSS3, JavaScript ES6+

### 1.3. Языки программирования
- PHP 7.4+ (серверная часть)
- HTML5 (клиентская часть)
- CSS3 (оформление)
- JavaScript (интерактивные элементы)
- SQL (работа с базой данных)

## 2. Функциональное назначение

### 2.1. Класс решаемых задач
Программа предназначена для решения задач обучения и тестирования сотрудников в области кибербезопасности, а именно:
- Обучение распознаванию фишинговых атак
- Тренировка правильных поведенческих реакций при обнаружении подозрительных веб-ресурсов
- Статистический анализ эффективности обучения

### 2.2. Назначение программы
Создание реалистичной имитации фишинговых сайтов для проведения учебных тренировок с последующим анализом действий пользователей и предоставлением развернутой обратной связи.

### 2.3. Функциональные ограничения
- Программа предназначена исключительно для учебных целей в контролируемой среде
- Не допускается использование для реальных фишинговых атак
- Требует настройки и администрирования квалифицированным специалистом
- Статистика сохраняется только при наличии подключения к СУБД

## 3. Описание логической структуры

### 3.1. Алгоритм работы программы
```
Начало
├── Пользователь получает доступ к тренажёру
├── Отображение имитации фишингового сайта
├── Варианты действий пользователя:
│ ├── Ввод данных в форму → Регистрация фишинговой попытки
│ ├── Уход через предупреждение → Регистрация успешного избегания
│ └── Закрытие страницы → Регистрация незавершенной сессии
├── Запись результатов в базу данных
├── Перенаправление на страницу обучения
├── Отображение статистики и рекомендаций
└── Конец
```


### 3.2. Используемые методы
- Сессионная аутентификация (PHP sessions)
- Объектно-ориентированное программирование (PDO для работы с БД)
- AJAX-технологии для динамического обновления статистики
- Адаптивная верстка (CSS Grid, Flexbox)
- Градиентный дизайн для создания привлекательного интерфейса

### 3.3. Структура программы

#### 3.3.1. Основные модули:
1. **Модуль симуляции фишинга** (`index.php`)
   - Генерация реалистичных фишинговых страниц
   - Обработка введенных пользователем данных
   - Логирование посещений

2. **Модуль антивирусного предупреждения** (`prewarning.php`)
   - Имитация систем безопасности
   - Запись реакции пользователей на предупреждения
   - Редирект в зависимости от выбора пользователя

3. **Модуль обучения** (`training_completed.php`)
   - Анализ действий пользователя
   - Отображение обучающих материалов
   - Сравнение фишинговых и легитимных сайтов

4. **Модуль статистики** (`stats.php`, `index.php`)
   - Сбор и агрегация данных
   - Визуализация результатов
   - Генерация рекомендаций

5. **Модуль управления данными**
   - Работа с СУБД PostgreSQL
   - Хранение учетных данных, посещений, статистики

#### 3.3.2. База данных:

```
phishing_training/
├── simulation_results (результаты симуляций)
├── legitimate_credentials (легитимные учетные данные)
├── av_warning_stats (статистика предупреждений)
└── training_stats (агрегированная статистика)
```

### 3.4. Связи с другими программами
- Интеграция с веб-сервером для обработки HTTP-запросов
- Взаимодействие с СУБД PostgreSQL через PDO
- Совместимость с современными веб-браузерами

## 4. Используемые технические средства

### 4.1. Аппаратные требования
- **Минимальные:**
  - Процессор: 2 ядра, 2 ГГц
  - ОЗУ: 2 ГБ
  - Дисковое пространство: 10 ГБ
  - Сетевое подключение: 10 Мбит/с

- **Рекомендуемые:**
  - Процессор: 4 ядра, 3 ГГц
  - ОЗУ: 4 ГБ
  - Дисковое пространство: 20 ГБ
  - Сетевое подключение: 100 Мбит/с

### 4.2. Программные платформы
- **Операционные системы:**
  - Linux (Ubuntu 20.04+, CentOS 8+)
  - Windows Server 2019+
  - macOS 10.15+ (для разработки)

- **Веб-серверы:**
  - Apache 2.4+ с mod_php
  - Nginx 1.18+ с PHP-FPM

## 5. Вызов и загрузка

### 5.1. Способ вызова программы
Программа вызывается через веб-браузер по URL-адресу:

**http://[адрес_сервера]/[путь_к_программе]/**

### 5.2. Входные точки
- `/index.php` - главная страница тренажёра
- `/prewarning.php` - страница антивирусного предупреждения
- `/training_completed.php` - страница результатов обучения
- `/stats.php` - страница статистики

### 5.3. Требования к памяти
- **ОЗУ:** 256 МБ на процесс PHP
- **Дисковое пространство для БД:** 100 МБ на 10 000 записей
- **Временные файлы сессий:** до 1 МБ на активного пользователя

## 6. Входные данные

### 6.1. Характер и организация входных данных
- **Пользовательские данные:**
  - Логин и пароль (вводимые в фишинговые формы)
  - Действия пользователя (клики, переходы)
  - Временные метки
  - IP-адреса и User-Agent

- **Конфигурационные данные:**
  - Параметры подключения к БД
  - Настройки сессий
  - Легитимные учетные данные для проверки

### 6.2. Формат и способ кодирования
- **Данные форм:** UTF-8, валидация на стороне сервера
- **Сессионные данные:** сериализованные массивы PHP
- **Данные БД:** JSON для хранения структур данных
- **Логи:** текстовый формат с временными метками

## 7. Выходные данные

### 7.1. Характер и организация выходных данных
- **Статистические отчеты:**
  - Общее количество посещений
  - Количество фишинговых попыток
  - Количество успешных избеганий
  - Процент эффективности обучения

- **Обучающие материалы:**
  - Информация о признаках фишинга
  - Рекомендации по безопасности
  - Визуальные сравнения

- **Технические данные:**
  - Логи работы системы
  - Дампы базы данных (по запросу)
  - Резервные копии конфигураций

### 7.2. Формат и способ представления
- **Веб-интерфейс:** HTML5 с CSS3-стилизацией
- **Диаграммы:** CSS-градиенты и прогресс-бары
- **Таблицы:** адаптивная HTML-разметка
- **Экспорт данных:** возможность экспорта в CSV/JSON
